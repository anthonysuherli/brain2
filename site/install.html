Capturing in 60 seconds.

Install the package, start the local API on 127.0.0.1, capture once — then come back to a populated resume card. No account, no key; the loopback backend keeps everything on your machine.

1

Install the package

The local tier additionally needs sqlite-vec for vector search on SQLite.

$ pip install brain2 sqlite-vec
2

Start the local API

The blessed launcher binds 127.0.0.1 and turns auth off — safe because it never leaves loopback.

$ BRAIN2_BACKEND=local python -m brain2.api.main
● serving on http://127.0.0.1:8000 — loopback only
3

Sanity-check the setup (optional)

Confirms the database, sqlite-vec, and the loopback bind before you wire anything up.

$ python -m brain2.api.main --check
● ✓ db   ✓ sqlite-vec   ✓ loopback
4

Wire it into Claude Code

Add the local dev marketplace, install the plugin, then reload the session so the MCP server connects.

$ /plugin marketplace add /path/to/brain2
$ /plugin install brain2@brain2

Then capture and resume from any session:

› /brain2:capture  ·  /brain2:resume
5

Or run the MCP server directly

For any MCP-compatible client or agent — exposes brain2_capture, brain2_resume, brain2_explore.

$ BRAIN2_BACKEND=local python -m brain2.interfaces.mcp.server
◌ The free tier is local-first and verifiable: SQLite at ~/.brain2/brain.db, bound to loopback, no account and no API key. Your code context never leaves your laptop.
◦ Cloud is designed, not yet built. brain2 launches free-first. The paid differentiators — cross-machine sync, cross-repo search, team sharing, managed keys — are on the roadmap, not shipping today.

Under the hood, the cloud path swaps the local SQLite store for a hosted Supabase backend behind the same get_store() seam, with a login for a real per-request token. When it's ready, the same capture→resume loop and the same surfaces will simply follow you across machines — no change to how you work.
