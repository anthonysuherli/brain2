1 · The problem & the product

Every interruption forces a developer to rebuild context — re-open files, re-read diffs, reconstruct “what was I doing?” That tax runs about 9.5 minutes per resume. br8n captures your working intent the moment you're interrupted and replays it as a 30-second resume card when you come back.

~9.5mcontext-rebuild tax per interruption (what br8n removes)
30starget time-to-resume with the resume card
2tiers from one engine — free local & paid cloud

br8n is a self-contained fork of the Delapan knowledge engine, repurposed from “research a topic” into “remember where I was.”

2 · The end-to-end loop

interruption return to work │ │ CAPTURE ───▶ persist as Finding ───▶ RESUME (tap) ───▶ coverage band (snapshot: in a repo+branch KB ├─ rich / sparse → resume card branch, files, └─ gap → EXPLORE (web gap-fill) diff, hypothesis) └──▶ refresh card
  1. Capture — a WorkspaceSnapshot (branch, open/cursor files, git diff --stat, and a one-line hypothesis) collected sub-second on blur / git_checkout / idle / manual.
  2. Persist — an adapter turns the snapshot into a Finding in a KB keyed by project = repo name, kb = git branch.
  3. Resume — tapping the KB returns a preamble (synopsis spine + latest hypothesis + recent snapshots) and a coverage band (rich / sparse / gap).
  4. Explore — on a gap, the pipeline (plan → search → crawl → extract → merge) seeds the KB from the web and rebuilds the synopsis; the card auto-refreshes.

3 · Architecture

All engine logic — capture, preamble/coverage banding, synopsis, exploration — is shared and never touches a storage client directly; it calls get_store(). A Store protocol has two implementations, selected by BR8N_BACKEND.

ConcernFree / localPaid / cloud
Store implSQLiteStoreSupabaseStore
PersistenceSQLite + sqlite-vec at ~/.br8n/brain.dbSupabase: GoTrue + pgvector + RLS
Identitysingle synthetic org_id="local"real JWT per request
Authnone — loopback-onlyBearer BR8N_API_KEY
The local launcher refuses to start on a non-loopback host because auth is disabled there — that guard is the entire reason the auth-less tier is safe.

Imports are always br8n.*; the fork is standalone with no cross-repo dependency on Delapan. Engine improvements are synced manually (copy file, rename imports).

7 · Getting-started user guide

Prerequisites

  • Python 3.11, git, and (local tier) sqlite-vec. uv is intended but plain venv works.
  • Cloud tier only: Supabase creds + a BR8N_API_KEY.

Step 1 — Install the backend

cd backend
python3.11 -m venv .venv                          # or: uv venv
.venv/bin/pip install -e ".[dev]" sqlite-vec      # or: uv sync && uv pip install sqlite-vec
cp .env.example .env                              # pick the FREE or PAID block

Step 2 — Run the engine

Free / local (SQLite, no Supabase, no API key, loopback-only):

BR8N_BACKEND=local python -m br8n.api.main            # blessed launcher; binds 127.0.0.1:8002
BR8N_BACKEND=local python -m br8n.interfaces.mcp.server

Paid / cloud (today's Supabase backend; needs the cloud .env block):

uvicorn br8n.api.main:app --reload --port 8002
python -m br8n.interfaces.mcp.server

Sanity check: curl http://127.0.0.1:8002/health

Step 3 — Install the Claude Code plugin

/plugin marketplace add /Users/suherli/Repositories/br8n
/plugin install br8n@br8n
Then reload the session so the MCP server connects. The root .mcp.json runs the server from backend/, so that dir needs br8n + deps installed.

Step 4 — Use it: the daily loop

WhenDo thisWhat happens
About to switch away/br8n:captureSaves a snapshot: branch + files + diff + your one-line hypothesis
Coming back/br8n:resumeReplays the 30-second resume card
Need an answer from this repo+branch's memory/br8n:search <q>Ground → grow → answer
Card says coverage=gap/br8n:explore <topic>Forces the web gap-fill pipeline, then refreshes the card

Targets resolve automatically: project = git repo name, kb = current git branch — no active-KB state to manage.

Step 5 — iOS companion (optional)

cd ios-app
xcodegen generate
open Br8n.xcodeproj   # then build & run in Xcode (needs full Xcode, not just CLT)

A native SwiftUI read spine: sign in, browse cross-repo activity, and read resume cards on the go against the same /v1 API.

The 30-second mental model: Capture when you leave → Resume when you're back. Free tier keeps everything on your machine; the cloud tier is the same engine with sync/sharing on the roadmap.
